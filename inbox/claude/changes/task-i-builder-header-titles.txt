WHAT WAS IMPLEMENTED
--------------------
A builder mode system that correctly differentiates between:
1. Template Mode - When user opens a template from Templates screen
2. Design Mode - When user opens a saved design from My Designs

HEADER BEHAVIOR
----------------
Template Mode:
- Main Title: Template name (e.g., "Partner Essentials NL")
- Subtitle: Hidden

Design Mode:
- Main Title: Project/Design name (e.g., "Q4 Email Campaign")
- Subtitle: Template name (e.g., "Partner Essentials NL")

HOW IT WORKS
------------
1. State Management:
   - Added window.builderMode flag ('template' | 'design')
   - Set when loading templates or designs
   - Persists across operations

2. Central Update Function:
   - updateBuilderHeader({ mode, templateName, designName })
   - Handles all DOM updates safely
   - Mode-specific logic for show/hide subtitle

3. Integration Points:
   - loadTemplateByKey() → sets mode='template'
   - loadDesignFromCard() → sets mode='design'
   - loadProjectFromComputer() → sets mode='design'
   - saveToMyDesignsWithFolder() → sets mode='design'
